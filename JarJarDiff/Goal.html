Goal

The goal of this project is to compare two versions of the same codebase and list the API differences. The tool lists:

  • added or removed packages
  • added or removed interfaces
  • added or removed classes
  • changes in deprecation of interfaces or classes
  • modifiled interfaces or classes
    • added or removed fields
    • added or removed constructors
    • added or removed methods
    • changes in deprecation of fields, constructors, or methods
    • modifiled fields, constructors, or methods

This information is important for conducting API quality control. API providers can use the tool to generated change reports for new versions of their APIs, letting customers know where they will need to spend their time when upgrading.

I used my Dependency Finder package for reading in the codebase and getting lists of packages, interfaces, classes, fields, constructors, and methods.
